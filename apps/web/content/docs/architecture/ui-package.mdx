## Components

<Tabs items={["Primitives", "Layout", "Common", "Landing", "Navigation"]}>
  <Tab value="Primitives">

15+ shadcn/ui components (`new-york` style) in the root of `src/components/`:

`avatar`, `breadcrumb`, `button`, `card`, `collapsible`, `dropdown-menu`, `input`, `label`, `scroll-area`, `select`, `separator`, `sheet`, `sidebar`, `skeleton`, `tooltip`

All built on Radix UI primitives except `card`, `input`, `breadcrumb` (native HTML), `skeleton` (CSS animation), and `sidebar` (custom compound component).

## Pages

Full page components consumed by both apps. The app routes are thin wrappers around these.

| Page | Route |
|---|---|
| `HomePage.tsx` | `/home` |
| `DashboardPage.tsx` | `/dashboard` |
| `SettingsPage.tsx` | `/settings` |
| `OverviewPage.tsx` | `/dashboard/overview` |
| `AnalyticsPage.tsx` | `/dashboard/analytics` |

This is why both apps can share the same UI: the page logic lives here, not in the apps.

## Hooks

| Hook | What it does |
|---|---|
| `use-mobile.ts` | Returns `true` below the mobile breakpoint |
| `use-mounted.ts` | Returns `true` after first client render |
| `use-theme-transition.ts` | View Transition API wrapper for smooth theme switching |
| `use-language-switcher.ts` | Locale change with theme re-application |

## Stores

Zustand stores with `localStorage` persistence where needed.

| Store | State |
|---|---|
| `greet-store.ts` | Greeting form input + response |
| `sidebar-store.ts` | Open/closed state, sidebar variant |
| `theme-store.ts` | Selected color theme (`data-theme` attribute) |

## Exports

Wildcard exports in `package.json`. No barrel files.

```json title="package.json (exports)"
{
  "./globals.css": "./src/styles/globals.css",
  "./postcss.config": "./postcss.config.mjs",
  "./components/*": "./src/components/*.tsx",
  "./providers/*": "./src/providers/*.tsx",
  "./pages/*": "./src/pages/*.tsx",
  "./pages/subpages/*": "./src/pages/subpages/*.tsx",
  "./hooks/*": "./src/hooks/*.ts",
  "./stores/*": "./src/stores/*.ts",
  "./config/*": "./src/config/*.ts",
  "./lib/*": "./src/lib/*.ts",
  "./scripts/*": "./src/scripts/*.ts"
}
```

```tsx
// Usage
import { Button } from "@workspace/ui/components/button";
import { AppLayout } from "@workspace/ui/components/layout/app-layout";
import { useThemeStore } from "@workspace/ui/stores/theme-store";
import { HomePage } from "@workspace/ui/pages/HomePage";
```

<Callout type="info">
Direct path imports (no barrel index). Tree-shaking works out of the box and circular dependency issues are avoided.
</Callout>

## Global Styles

```css title="packages/ui/src/styles/globals.css"
@import "tailwindcss";
@import "tw-animate-css";
@import "./themes.css";

@source "../../../apps/**/*.{ts,tsx}";
@source "../**/*.{ts,tsx}";

@custom-variant dark (&:is(.dark *));
```

The `@source` directives tell Tailwind to scan both the UI package and all apps for class usage. Both apps import this stylesheet via `@workspace/ui/globals.css`.

<Callout type="warn">
Don't create separate Tailwind entry points in the apps. Use the shared `globals.css` so all workspaces share the same design tokens and utilities.
</Callout>
