## Web

| Technology | Version | Role |
|---|---|---|
| [Next.js](https://nextjs.org) | 16 | App Router, SSR for web, static export for native |
| [React](https://react.dev) | 19 | UI rendering |
| [TypeScript](https://www.typescriptlang.org) | 5.9 | Strict type safety across every package |

Next.js handles both sides of this stack. The web app runs with SSR and middleware. The native app uses `output: "export"` to produce static HTML that Tauri loads into a webview. Same framework, different output modes.

## Native

| Technology | Version | Role |
|---|---|---|
| [Tauri](https://tauri.app) | 2 | Desktop and mobile runtime (Rust core + system webview) |
| [Rust](https://www.rust-lang.org) | stable | Native backend, IPC commands |
| [serde](https://serde.rs) | 1 | JSON serialization for IPC payloads |

Tauri delegates rendering to the OS-native webview instead of bundling Chromium, significantly reducing the final binary footprint.

## Styling

| Technology | Version | Role |
|---|---|---|
| [Tailwind CSS](https://tailwindcss.com) | 4 | Utility-first CSS with `@theme` and native CSS variables |
| [shadcn/ui](https://ui.shadcn.com) | latest | Copy-paste Radix UI components, `new-york` style |
| [Radix UI](https://www.radix-ui.com) | various | Accessible, unstyled primitives (`dialog`, `dropdown-menu`, `select`, etc.) |
| [Motion](https://motion.dev) | 12 | Animations (landing page transitions, staggered reveals) |
| [Lucide](https://lucide.dev) | 0.577 | Icon set |
| [cmdk](https://cmdk.paco.me) | 1.1 | Command palette (⌘K menu) |
| [Vaul](https://vaul.emilkowal.ski) | 1.1 | Drawer component |
| [Sonner](https://sonner.emilkowal.ski) | 2 | Toast notifications |

Tailwind v4 dropped the JS config file. Themes are now pure CSS custom properties scoped by `data-theme` attributes, which drives the dynamic theme architecture.

## Internationalization

| Technology | Version | Role |
|---|---|---|
| [next-intl](https://next-intl.dev) | 4.9 | Routing, middleware, `useTranslations`, message loading |

Wrapped in `@workspace/i18n` so both apps share the same locale config, messages, and navigation helpers. Type-safe keys via module augmentation.
